# creation-info-card — 信息卡片生成

> 文案 → Twitter 风格多页卡片图片（3:4 竖屏，适合小红书图文）

## 触发词
- "做卡片" / "信息卡片" / "info card" / "图文卡片"

## 设计风格

Twitter/X 推文截图风格：
- 白色背景
- 圆形头像 + 昵称 + 日期
- 正文黑色，干净排版
- 右上角页码（多页时显示）
- 无多余装饰，朴素简洁

## 输入

| 项目 | 必填 | 说明 |
|------|------|------|
| 文案 | 是 | Markdown 文本，# 标题行渲染为大号粗体（首页），`【】` 渲染为粗体小标题，正文用正常字号，支持 `![](path)` 内嵌图片 |
| 头像图片 | 是 | 任意图片，自动裁切为圆形 |
| 昵称 | 是 | 显示名（如"玛莎Dojo"） |
| 日期 | 否 | 显示在昵称下方（如"2026年3月16日"），默认当天 |

## 输出

- `card-1.png` ~ `card-N.png`（3:4 竖屏，1080x1440）
- 存储在对应话题的 `creation/` 目录下

## 创作流程

### 1. 文案撰写

风格要求：
- **朴素简单**，用个人感想的口吻，不是新闻稿
- **不用表情符号**，不用小红书体
- **1500～2000 字为佳**，不要过度压缩原始素材（低于 1000 字会导致关键论点和例子丢失，读起来像摘要而非文章）
- 不需要在末尾标注数据来源和引用
- 从精采素材（sources.json + articles/）或旁白稿中提取具体数据点和引用

文案结构要求：
- 用 `【章节标题】` 语法划分段落（独占一行），渲染为粗体小标题（STHeiti 38px）
- 章节标题前自动增加额外间距（1倍行高），视觉上清晰分隔
- 章节接近页面底部（>75%）时自动换到下一页，避免标题孤悬
- 每个章节内包含 1-3 个自然段，保持阅读节奏
- 结尾段简短有力，作为整篇的收束

文案结构参考（不固定，按话题调整）：
- 开头：一个具体的数据或事实引入
- 中段：按逻辑分 3-5 个章节展开，每个章节有小标题
- 转折：不同角度的观点
- 结尾：个人判断，简短有力

### 2. 卡片生成

Python 脚本，依赖 Pillow。

#### 参数配置

```python
CARD_W, CARD_H = 1080, 1440   # 3:4 竖屏
BG_COLOR = "#ffffff"
TEXT_COLOR = "#0f1419"
GRAY_COLOR = "#536471"
AVATAR_SIZE = 96
PADDING = 72
```

#### 核心逻辑

1. **圆形头像**：读取原图 → 中心裁切正方形 → 缩放到 96px → 圆形 mask
2. **文字换行**：逐字符测量宽度（支持中英文混合），超过 max_width 时换行
3. **章节标题**：`【标题】` 语法（独占一行），渲染为 STHeiti 38px 粗体，前方增加 1 倍行高间距，接近页底（>75%）时自动换页
4. **内嵌图片**：`![](path)` 语法，图片缩放到 936px 宽（保持比例），12px 圆角，居中显示
5. **自动分页**：按块（段落、章节标题或图片）为单位分配到页面，一块放不下时自动新起一页，图片不跨页拆分
6. **排版**：头像+昵称+日期在顶部，正文从下方开始，行距 1.6 倍，段间距 0.5 倍行高
7. **macOS 兼容**：Pillow 保存 PNG 后用 `sips -s format png` 后处理，修复 Preview 打开问题

#### 字体

macOS:
- 标题：`/System/Library/Fonts/STHeiti Medium.ttc`（42px，粗体，仅首页）
- 章节标题：`/System/Library/Fonts/STHeiti Medium.ttc`（38px，粗体）
- 昵称：`/System/Library/Fonts/STHeiti Medium.ttc`（37px）
- 正文：`/System/Library/Fonts/Hiragino Sans GB.ttc`（37px）— PingFang 标点位置偏高，Hiragino 标点位置正确
- 辅助文字（日期/页码）：`/System/Library/Fonts/PingFang.ttc`（30px / 28px）

#### 图片插入

在 Markdown 文案中用标准图片语法插入图片：

```markdown
文字段落...

![](../output/scene1-light.png)

下一段文字...
```

图片行为：
- 图片路径相对于 draft 文件所在目录解析，也支持绝对路径
- 缩放到 936px 宽度（保持宽高比），超高图片裁切到页面可用高度内
- 12px 圆角（alpha mask）
- 水平居中显示
- 图片不跨页拆分，放不下则整块移到下一页
- 图片文件不存在时打印警告并跳过

#### 分页算法

```
每页可用高度 = 卡片高度 - 顶部(头像区) - 底部留白(40px)
逐块（段落或图片）累加高度，超出时换页
段落和图片都不跨页拆分（保持语义完整）
```

#### 注意事项

- 最后一页如果内容太少（不到可用高度 30%），把前一页末尾的文字块匀过来（不移动图片块）
- 页码格式：`1/3`，右上角灰色小字
- 日期放在昵称下方（替代 @handle 的位置），不放底部
- macOS Preview 兼容：Pillow 生成的 PNG 需 `sips -s format png` 后处理才能正常打开
