# 精简

> 分析内容结构 → 识别冗余 → 生成删减方案

## 快速使用

```
用户: 帮我精简这个视频到10分钟
用户: 分析一下哪些内容可以删
用户: 视频太长了，帮我删减
```

## 适用场景

| 场景 | 说明 |
|------|------|
| 教程视频 | 删除跑题、重复讲解、过度铺垫 |
| 课程录制 | 压缩到目标时长，保留核心内容 |
| 口播剪辑 | 在口误处理之外，进一步精简内容 |

## 流程

```
1. 读取转录稿（字幕稿或 transcript.json）
    ↓
2. 分段梳理内容结构
    ↓
3. 标记每段：主题、时长、保留建议
    ↓
4. 识别可删减内容
    ↓
5. 计算目标时长差距
    ↓
6. 生成删减方案（优先级排序）
    ↓
【用户确认】→ 用户确认后，执行 /videocut:剪辑
```

## 一、内容结构分析

### 分段原则

| 维度 | 说明 |
|------|------|
| 按主题切分 | 每个独立话题一段 |
| 标记时间段 | `(00:00-01:17)` 格式 |
| 估算时长 | 精确到秒 |

### 输出格式

```markdown
## 内容结构分析

| 时间段 | 主题 | 时长 | 保留建议 |
|--------|------|------|----------|
| 00:00-01:17 | 开场引入 | 77s | 精简到30s |
| 01:17-02:27 | 功能概述 | 70s | 保留核心 |
| 02:27-04:00 | 详细演示 | 93s | 可删除 |
```

---

## 二、删减类型

### A. 完全删除

| 类型 | 示例 | 判断标准 |
|------|------|----------|
| 跑题内容 | 讲安装时聊产品对比 | 与主题无关 |
| 重复讲解 | 同一概念讲两遍 | 信息冗余 |
| 过度铺垫 | 3分钟才进入正题 | 开场太长 |

### B. 精简内容

| 类型 | 示例 | 处理方式 |
|------|------|----------|
| 废话 | "就是说"、"然后呢" | 删除填充词 |
| 吐槽 | 抱怨某功能难用 | 删除情绪表达 |
| 解释过多 | 小白不需要的技术细节 | 精简到核心 |

### C. 技术处理（与口误识别合并）

| 类型 | 处理方式 |
|------|----------|
| 静音段 | 由 `/videocut:剪口播` 识别 |
| 口误 | 由 `/videocut:剪口播` 识别 |
| 语气词 | 由 `/videocut:剪口播` 识别 |

## 四、工作流整合

### 完整流程

```
/videocut:剪口播  → 口误/静音/语气词识别
        ↓
/videocut:精简    → 内容分析 + 删减建议  ← 可选
        ↓
【用户确认删除清单】
        ↓
/videocut:剪辑    → 执行删除
        ↓
/videocut:字幕    → 添加字幕
```

### 与剪口播的关系

| 维度 | 剪口播 | 精简 |
|------|--------|------|
| 关注点 | 技术问题（口误、静音） | 内容问题（冗余、跑题） |
| 自动化 | 高（规则识别） | 低（需理解语义） |
| 用途 | 必做 | 可选（时长超标时用） |

## 五、注意事项

### 删减优先级

1. **跑题内容** - 优先删除，对主线无影响
2. **重复讲解** - 保留更清晰的版本
3. **过度铺垫** - 精简而非完全删除
4. **细节解释** - 根据目标受众判断

### 保留原则

- 核心操作步骤不能删
- 关键概念解释保留一次
- 用户可能的疑惑点保留解答
