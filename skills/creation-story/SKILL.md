# creation-story — 虚构故事生成

> 热点素材 → 第一人称虚构故事（~5000字，分章节）

## 触发词
- "写故事" / "虚构故事" / "story gen" / "小说"

## 输入

| 项目 | 必填 | 说明 |
|------|------|------|
| 话题素材 | 是 | 精采的 sources.json + articles/，提供真实数据点和背景 |
| 字数要求 | 否 | 默认 ~5000 字 |
| 章节数 | 否 | 默认 5-8 章 |

## 输出

- `story.md`（Markdown，含一级标题 + 二级标题章节）
- 存储在对应话题的 `creation/` 目录下

## 写作风格

### 核心原则
- **第一人称**，虚构主角，不是作者本人
- **朴素简单**，不堆砌辞藻，不用表情符号
- **基于真实**，故事虚构但引用的数据、人物、事件是真实的
- **不说教**，通过主角的经历和感受传达观点，不在结尾升华总结

### 叙事结构参考

```
章节 1: 建立 — 主角的日常，建立读者认同感
章节 2-3: 发展 — 变化开始出现，主角感知到压力
章节 4: 转折 — 关键事件或对话，主角面临选择
章节 5-6: 应对 — 主角做出选择，面对新的现实
章节 7: 结尾 — 开放式结局，不给标准答案
```

不固定，按话题调整。核心是有清晰的情感弧线：从安稳 → 不安 → 危机 → 选择 → 新平衡。

### 语言

- 短句为主，长句不超过 40 字
- 对话要像真人说话，不要书面腔
- 细节具体：写"望京一栋不太新的写字楼"，不写"某个城市的某栋楼"
- 允许出现真实人名和引用（Karpathy、Sam Altman 等说过的话）
- 每章 600-800 字，节奏紧凑

### 禁忌

- 不用表情符号
- 不用"震惊""颠覆""细思极恐"等营销词
- 不在结尾写鸡汤式总结
- 不用排比句堆气势
- 不写全知视角的评论，所有信息通过主角的视角呈现

## 创作流程

### 1. 选题评估

适合写虚构故事的话题特征：
- 有**个人层面的影响**（职业、生活方式的改变）
- 有**情感张力**（焦虑、选择、适应）
- 读者**能代入**（不是纯技术或纯商业话题）

不适合的：纯产品发布、纯技术架构、纯行业分析

### 2. 素材提炼

从精采素材中提取：
- **真实数据点**：用于故事背景（如"招聘减少20%"、"裁员6万人"）
- **真实人物引用**：用于对话或主角看到的新闻（如 CEO 的原话）
- **真实事件**：用于推动情节（如 OAuth 宕机导致 Agent 实验室瘫痪）

### 3. 故事撰写

直接写成完整的 Markdown 文件：
- 一级标题：故事标题
- 二级标题：章节标题（"一、xxx" 格式）
- 正文：纯文本，无 Markdown 格式标记

## 文件位置

|------|------|
